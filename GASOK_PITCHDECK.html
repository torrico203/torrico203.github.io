QUARTER TURN1 / 9
Problem

Web3 대중화의 병목은 기술이 아니라 온보딩이다

  • 시드 문구 · 가스비 · 지갑 익스텐션 — 일반인은 첫 화면에서 이탈한다 "지갑을 만드세요"가 등장하는 순간 대중 서비스가 아니게 된다
  • 기존 Web3 게임: 크립토 유저를 게임으로 데려오는 방향 — 이미 지갑이 있는 사람들끼리의 시장 우리는 반대로 간다: 이미 놀고 있는 게임 유저를 크립토로
  • 국내 게임 × 크립토 피치 대부분은 P2E 환전 금지 규제에 걸려 시작부터 좌초 우리는 환전 배제를 설계 원칙으로 삼아 이 리스크를 차별화로 뒤집는다
QUARTER TURN × GIWA2 / 9
Solution

게임을 했을 뿐인데, 온체인 자산과 기록이 생긴다

LAYER 01

보이지 않는 지갑

포털 소셜 로그인 시 GIWA 스마트 계정 자동 생성. 시드 문구 없음, 가스는 페이마스터 대납. "지갑 만들기"라는 단계 자체가 UX에서 사라진다.

LAYER 02 — 핵심

시즌 랭킹 명예의 전당

가동 중인 일간/주간 리더보드를 시즌제로 확장. 시즌 마감 → 상위 랭커에게 티어별 뱃지 SBT 자동 민팅 → 온체인 영구 박제. 시즌마다 반복되는 온체인 이벤트.

LAYER 03

스테이블코인 온램프

GIWA 스테이블코인 → 게임 재화 단방향 충전. 인출 경로는 의도적으로 없음 — 결제 수단 추가일 뿐 P2E가 아니다.

  • 뱃지는 양도 불가(ERC-5192 계열) + 무가치 — 보상은 "명예의 박제"까지만 무가치 SBT라 치팅의 금전 유인 자체가 구조적으로 낮다
  • 업적 SBT를 더해 GIWA 생태계 전체에서 참조 가능한 크로스게임 게이머 프로필 구성
QUARTER TURN × GIWA4 / 9
Regulation-Compatible by Design

규제 리스크를 셀링 포인트로

설계 결정근거
게임 재화(다이아)는 오프체인 유지 — 토큰화하지 않음 재화의 시장 거래 가능성 원천 차단
업적·시즌 뱃지는 양도 불가 SBT + 무가치 환전·경품 조항 비해당
랭킹 보상으로 유가 토큰을 지급하지 않음 "경쟁 → 환금 보상" 루프 차단 — P2E 재해석 여지 제거
크립토 → 재화 단방향만 허용, 인출 경로 부재 게임산업법 환전 금지의 회피가 아닌 준수

2026 게임산업법 전면개정안에서도 환전 금지는 유지 전망. P2E성 피치가 규제 벽에 걸릴 때, 처음부터 규제-호환으로 설계된 팀이라는 포지션.

MVP Scope — GIWA Sepolia

2라운드 요구물에 1:1 대응하는 실행 계획

  • 시즌 뱃지 / 업적 SBT 컨트랙트 GIWA Sepolia 배포 → 퍼블릭 스마트컨트랙트 링크 요건 충족
  • 시즌 확정 데모 (핵심): 주간 랭킹 → 서버 검증 → 시즌 마감 → 배치 민팅 → 명예의 전당 온체인 조회
  • 임베디드 지갑 플로우: 소셜 로그인 → 스마트 계정 생성 (가스 대납)
  • 테스트넷 스테이블코인 → 재화 충전 데모 시간 부족 시 아키텍처 문서로 대체
  • 접속 가능한 사이트: 운영 중인 포털 그 자체 데모 영상 + 라이브 URL 동시 제출

공개 코드는 신규 저장소(컨트랙트 + 연동 레이어)로 격리 — 시크릿 제로 원칙, 기존 게임 IP는 비공개 유지.
