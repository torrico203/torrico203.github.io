핵심 강점

  • ① 1인 완결 실행력 기획 → 개발 → 런칭 → 운영을 혼자서 완결. QUARTER TURN 은 개발 2개월 만에 게임 15종 등록(완성 5종), 통합 지갑 SDK, Firebase 백엔드, ko/en i18n 까지 1인 구축.
  • ② HTML5 게임 · 플랫폼 10년 WebGL 게임 엔진·HTML5 게임 프레임워크 국책과제 수행, HTML5 게임 허브(ROR) 채널링 운영, HTML5 상용 게임으로 일 매출 7천만 원 달성. 웹에서 게임을 서비스하는 일을 10년째 하는 중.
  • ③ 통합 지갑 아키텍처 QUARTER TURN 의 화폐(dia)는 호스트가 발행·소유하고 게임은 SDK 로만 차감하는 단일 권위 구조. 지갑 백엔드를 무중단 교체한 실증(localStorage→Firestore, 게임 코드 변경 0건)이 있어 온체인 지갑·결제 연동에 바로 대응 가능한 설계.
  • ④ 사업 · 글로벌 런칭 경험 퍼블리싱·채널링 계약 13건(유럽·러시아·일본·인도 등), 투자 유치 2회(시드·Pre-A), 게임 수출 10만 유로. 만들기만 하는 게 아니라 계약하고 서비스하고 정산까지 해 본 경험.

프로젝트 — QUARTER TURN

브라우저에서 바로 실행되는 HTML5 게임 플랫폼. 호스트가 계정·지갑·세이브·랭킹을 소유하고, 게임은 iframe + postMessage SDK 로 연결되는 호스트-게임 분리 아키텍처. 게임 15종 등록 (완성 5종 공개 서비스), 통합 화폐 dia, Unity·웹 레거시를 되살리는 리마스터 파이프라인까지 전부 1인 개발.

  • 지갑 단일 권위 — 화폐 발행은 호스트 전용, 게임은 SDK 로 차감만. 백엔드 교체(localStorage→Firestore)를 게임 코드 변경 0건으로 완료한 구조라, 온체인 지갑 연동 시에도 게임들은 그대로.
  • 확장 파이프라인 — 신규 게임 스캐폴딩 CLI 와 등록 검증 스크립트로 게임을 계속 얹는 구조. 리마스터·신작 9종 진행 중.
  • 로드맵 — 개발중 게임 완성·공개, 실결제 연동, 서버 권위 전환, 자체 도메인 연결 후 정식 런칭.
