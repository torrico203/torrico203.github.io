QUARTER TURN — 기술 상세

HTML5 게임 플랫폼의 호스트-게임 분리 아키텍처, dia 통합 화폐 SDK, 레거시 리마스터 파이프라인 노트

2026.05 — 현재 · quarterturn.pages.dev ↗
한국어 전용 · Korean only
육아휴직 기간에 진행 중인 1인 개발 프로젝트. 브라우저에서 바로 실행되는 HTML5 게임 플랫폼으로, 호스트가 계정·지갑·세이브·랭킹을 소유하고 게임은 iframe 으로 연결되는 구조. 현재 등록 게임 15종 (공개 11종 — 완성 5 · 개발중 6) 이 같은 SDK 규약 위에서 동작하며, Unity·웹 레거시 리마스터 파이프라인과 신작 개발을 병행.

1. 호스트-게임 분리 아키텍처

구조 개요

  • 게임마다 저장·화폐·랭킹을 따로 구현하는 대신 호스트가 계정·지갑·세이브·랭킹을 단일 소유하고, 게임은 정해진 SDK API 만 호출하는 구조로 설계. 게임 수가 늘어도 백엔드 결정은 호스트 한 곳에서만 발생.
  • 게임은 iframe 으로 마운트되고 호스트와는 postMessage 로만 통신 (src/lib/walletBridge.js). 요청마다 reqId 를 부여한 비동기 왕복 구조라 백엔드가 동기 → 비동기로 바뀌어도 게임 측 코드는 무변경.
  • iframe ↔ 호스트 왕복 실측 약 16ms. 결제 팝업(autoShop)이 브라우저 user-gesture 컨텍스트를 유지하는지 popup blocker 환경에서 검증.

Pakedev SDK public/vendor/pakedev-sdk.js (818 LOC)

  • 게임 개발자에게 노출되는 유일한 인터페이스. Pakedev.wallet (잔액 조회/차감/변경 구독), Pakedev.storage (키 단위 세이브), Pakedev.ranking (점수 제출), Pakedev.shop (충전 진입) 4개 네임스페이스로 구성.
  • 차감은 게임, 충전은 호스트 전용 — 화폐 발행 경로를 호스트 한 곳으로 고정해 서버 권위 전환 시 변경 지점을 단일화.
  • SDK 부재 환경(게임 단독 실행)용 dev 폴백을 규약에 포함 — 게임 개발자는 호스트 없이 자기 게임만 띄워 놓고 작업 가능.

2. 게임 라인업

포털 매니페스트(data/gameData.js) 기준 15종 등록. 완성 5종은 공개 서비스, 개발중 6종은 비밀번호 소프트 게이트로 잠금, 4종은 출시 스코프 조정으로 비공개 전환. 별도로 매니페스트 미등록 파이프라인 3종 진행 중.

완성 · 공개 서비스 (5종)

  • DROP POP — 공을 발사해 숫자 블록을 깨는 물리 벽돌깨기 (matter-js). 공 14종 수집, 하이스코어 랭킹.
  • Gemstone Match — 500 스테이지 3매치 퍼즐. 특수 블록 조합, 스테이지 랭킹.
  • 엽전맞고 — 흔들·폭탄·뻑·고도리 정식 룰 기반 스테이지식 맞고. 난이도 3종 × 6단계, 엽전을 걸고 상대를 오링시키는 진행 구조.
  • Flappy Block — 블록을 부수며 나는 원버튼 아케이드. 캐릭터 60종 수집·강화, 스테이지/거리 2개 랭킹 보드.
  • COSMIC DEFENCE — 5종 함선 덱으로 침공을 막는 우주 타워디펜스. PvP·협동 듀얼 모드, 합성·뽑기·배틀패스, 150ms 턴 단위 결정론 설계.

개발중 · 비밀번호 게이트 (6종)

  • BESTIARY — 몬스터를 사냥해 도감을 채우는 핵앤슬래시 수집 RPG (맵 317 · 스킬 72 · 펫 27). 채팅·프레즌스용 실시간 백엔드를 Cloudflare Workers + Durable Objects 로 분리 구축 — egress 무과금 특성을 이용해 실시간 기능의 운영 비용을 절감.
  • RELIQUARY — 서버 의존형 웹 MMO 레거시를 서버리스 싱글 구조로 재설계한 클린 리마스터. 유닛·아티팩트 수집과 던전·미궁 공략.
  • RE:roll Chess — 헥사 보드 오토배틀러. PvE 캠페인 + 비동기 PvP(패왕전), 캐릭터 72종.
  • Flip and Fight — 뒤집기와 전투를 결합한 세로형 캐주얼 로그라이크. 라인업 중 유일한 Phaser 3 프로젝트이자 신규 오리지널.
  • BANE & BOON — 저주를 받아들일수록 강해지는 던전 액션 로그라이크, 캐릭터 18종. 권리 정리 진행 중으로 코드네임 운영.
  • OUTBREAK — 실시간 액션 좀비 RPG, 640 스테이지, 장비·펫·스킬 수집. 권리 정리 진행 중으로 코드네임 운영.

파이프라인 · 매니페스트 미등록 (3종)

  • 괴력난신 — 한국 신화·민속 테마 오브매치 RPG (화·수·뇌·귀 4속성). 2018년 자사 원작의 기획서·리소스 기반 리마스터.
  • Hero404 — 탑다운 액션 로그라이트 RPG. Unity 2022 원본을 PixiJS 로 이식하는 UNITY-REMASTER-PLAYBOOK 의 적용 케이스.
  • 신규 오토배틀러 1종 — 기획 단계. RE:roll Chess 의 캐릭터 72종 자산을 재사용하는 설계.

3. dia 통합 화폐와 백엔드 교체

  • Phase 1 — 클라이언트 localStorage 권위로 지갑·거래내역·HUD·충전 페이지까지 선구현하여 UX 와 API 계약을 확정.
  • Phase 3 — Firebase Anonymous Auth + Firestore (asia-northeast3) 로 전환. wallets/{uid} 는 onSnapshot 라이브 구독, 거래는 transactions/{uid}/items/{txId} 에 txId 를 문서 ID 로 사용해 멱등성 보장, 세이브는 saves/{uid}/games/{gameId} 게임당 1문서에 키 단위 merge / deleteField.
  • 전환 결과 게임/SDK 코드 변경 0건 — wallet.js 가 공개 API 의 동기 시그니처를 유지한 채 내부에만 Firestore 미러를 추가. 브리지가 처음부터 비동기라 게임 측 영향 없음. HUD·상점·브리지 전부 무수정 통과.
  • 전체 store 에 DB 미설정 시 localStorage 폴백 유지 — 설정만으로 오프라인/로컬 모드 동작. prod 전환은 Cloudflare Pages 환경변수 (VITE_FIREBASE_*) 스왑만으로 수행, 코드 분기 없음.
  • Firestore Security Rules 를 저장소에서 수기 관리 (firebase deploy --only firestore:rules), 양성(본인 지갑 읽기 허용)·음성(타인 지갑 읽기 차단) 스모크 테스트로 규칙의 실제 강제 여부 검증.

4. 레거시 리마스터 파이프라인

  • 문제 정의: 되살릴 원본 다수가 소스·에셋만 남고 빌드 불가 상태 (엔진 버전 불일치, 유료 플러그인, 서비스 종료된 SDK). 원본 재빌드 대신 이식 절차를 표준화하는 방향으로 해결.
  • 절차 문서화 (UNITY-REMASTER-PLAYBOOK.md, REMASTER_PROMPT.md): 원작을 단일 진실 공급원(SSOT)으로 규정 — 스탯·가격·확률·UI 배치를 추측으로 재구현하지 않고 legacy 소스 grep/덤프에서 확인하는 것을 첫 번째 규칙으로 명문화.
  • 추출 전부 스크립트화: Unity *.prefab (UI 앵커/좌표/스프라이트), *.cs (전투식·비용 공식·스폰), 로컬라이제이션 테이블, BGDatabase 바이너리(*.bytes) 파서까지 직접 구현해 src/data/extracted/*.json 생성. 재실행 시 동일 결과 보장. export 담당자 부재로 사장됐던 밸런스 수치를 바이너리에서 자체 복구한 것이 첫 성과.
  • 프로젝트 표준 구조: reMasterPlan.md (계획 SSOT) + legacy/ (읽기 전용 원본) + devNote/decisions/ (ADR).
  • 이식 타깃 표준화: TypeScript strict + PixiJS v8 + DOM UI — 렌더링은 Pixi, UI 레이아웃은 웹 표준(DOM) 으로 분리.
  • 검증 가능한 단위로 분할 커밋 — 한 커밋 = 한 기능 + 실제 게임 화면 검증. 현재 리마스터·신작 9종이 이 절차로 진행 중.

5. 랭킹 무결성 · 운영 게이트

  • 검증가능성 클래스 분류: 게임별 치트 표면을 감사해 랭킹 보드를 등급화 (docs/INTEGRITY_AUDIT.md → docs/RANKING_AND_INTEGRITY.md). 서버 검증이 깨끗한 progression 보드(도달 스테이지 등)를 우선 적용하고, 점수성 보드는 서버 권위 전환 전까지 "표시하되 신뢰하지 않음" 등급으로 명시.
  • 세이브 변조 위협 모델 문서화 — 호스트 단일 지점 서명, 거래 내역 기반 보유 재구성, 비정상 점프 감사 추적. 실행 일정은 서버 권위 전환과 연동.
  • 런칭 게이트 (docs/LAUNCH_CHECKLIST.md): 개발 편의 장치 (초기 dia 자동 지급, 무료 충전, 전역 디버그 객체)를 파일:라인 단위로 목록화하고 항목별 제거 선후관계를 명시.
  • 권리 정리 게이트: 권리 확인이 끝나지 않은 타이틀은 코드네임 격리 + locked: true 소프트 게이트로 공개 목록에서 차단. 게임 내부 문자열·탭 제목의 구명 노출까지 제거. 정책을 데이터 필드로 만들어 배포 파이프라인이 강제하는 구조.
  • 규제 선행 검토: 가상화폐(dia)의 선불전자지급수단 해당 여부·환불 규정·등급분류·PG 이슈를 실결제 연동 전 상담 체크리스트로 정리 (docs/LEGAL_CONSULT_CHECKLIST.md).
  • AI 운영 보조 권한 경계 (docs/AI_STAFF.md): 읽기·분류·요약·초안 작성은 자동화하되, 공개 포스팅·제재·결제·외부 발송은 사람 승인 게이트로 분리. 공개용 산출물에는 [DRAFT — 사람 검토 대기] 머리표를 강제.

6. 구현 메모

  • 호스트: Vite MPA (vanilla JS) — 홈 / 게임뷰 / 상점 3개 엔트리, 약 3,700 LOC. Cloudflare Pages 배포.
  • Phaser 3 를 public/vendor/ 공유본으로 두고 iframe 간 HTTP 캐시로 재사용 — 게임별 중복 번들 제거.
  • 게임이 자체 FIT 스케일 미구현인 경우에만 호스트가 baseViewport 고정 크기 + transform scale 로 대응. 게임 측 FIT 구현 재납품 시 예외 제거.
  • 모바일에서 게임 화면 스케일 유동·가상 키보드로 인한 뷰포트 축소를 게임뷰 레이어에서 차단.
  • ko/en 런타임 i18n, PWA manifest, 신규 게임 스캐폴딩 CLI (npm run new-game) 와 등록 검증 스크립트 (validate-games).
현재 상태 (정직 노트)
  • 정식 서비스가 아니라 공개 개발 중. 공개 11종 중 완성 5종, 나머지 6종은 비밀번호 게이트 뒤의 개발 빌드. 런칭 체크리스트가 미완이라 개발 편의 장치가 코드에 남아 있음.
  • DAU·매출·리텐션 지표 없음 — 실결제 연동·마케팅 전 단계. 이 프로젝트의 평가 포인트는 트래픽이 아니라 아키텍처 결정과 그 근거.
  • 랭킹은 서버 검증 전으로 순위의 신뢰성을 주장하지 않음 (5장 분류 참조).
