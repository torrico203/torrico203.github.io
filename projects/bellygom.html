1. 크로스 플랫폼 추상화 전략

단일 진입점 추상화 (PlatformManager 싱글톤)

src/managers/PlatformManager.ts — 빌드타임 (VITE_TARGET_ENV) → 런타임 (navigator.userAgent) 2-tier 폴백 전략. 환경변수가 우선이라 트리쉐이킹과 정적 분석이 가능하고, build:web 으로 빌드된 번들도 모바일 웹/PC 를 자동 분기. 21개 파일에 걸친 105곳의 분기 처리가 모두 이 단일 소스에서 출발.

입력 추상화 (Adapter 패턴)

src/systems/InputSystem.ts — TV 리모컨 (keyCode 461 BACK, numpad 97-102), 키보드, 포인터(터치/마우스) 를 하나의 이벤트 (shootRequested, confirmActionRequested, backPressed) 로 통일. 화면 좌표를 발사 각도로 환산하는 updateAimAngle 은 X축은 그대로, Y축에 최소 거리 (40px) 를 강제해 터치/마우스 vs 키보드 회전 입력을 동일한 aimAngle 상태머신으로 흡수.

TV 전용 포커스 시스템 (Bridge 패턴)

src/managers/FocusManager.ts — IScrollableScene 인터페이스 + 타입 가드 isScrollable(scene) 로 씬마다 다른 스크롤 동작을 제네릭하게 위임. UP/DOWN/ENTER/SPACE 4개 키만으로 Container | Text | Image 어떤 GameObject 든 포커스 이동 가능. confirm 이벤트를 발행해 버튼 객체가 자체 클릭처럼 처리하도록 추상화.

결제 / 광고 다형성

  • src/managers/AdManager.ts: @capacitor-community/admob (Android) ↔ @apps-in-toss/web-framework 의 GoogleAdMob (Toss) 을 단일 showRewardVideoAd(): Promise<boolean> API 로 캡슐화. Toss 는 콜백 기반 (onEvent/onError), Android 는 listener handle 기반인데 Promise 로 모두 통일.
  • src/managers/PaymentManager.ts: cordova-plugin-purchase (Google Play) ↔ Toss IAP 두 결제 SDK 를 동일한 purchase(productId) 인터페이스로 통합. coinPkg.tossId 필드로 Google ID ↔ Toss SKU 매핑 (priceConfig.ts:43-56).

TypeScript Generic / Interface 활용

  • src/utils/GameObjectPool.ts:5-18: Phaser.GameObjects.GameObject & Visible 컴포넌트를 교집합 타입으로 강제, 풀 재사용 시 setActive/setVisible 안전 호출 보장.
  • src/interfaces/PlayerData.ts: keyof PlayerItems 를 Record 로 활용해 가격표 누락을 컴파일타임에 차단.
  • src/scenes/ShopScene.ts:23 — 동적 ID 리스트로 만든 type predicate 로 런타임/컴파일 타입 동기화.

2. 저사양 / 제한적 환경 최적화 (webOS & Toss)

TV 한정 렌더 다운그레이드

src/game.ts — Phaser 설정에서 maxTextures: 1, target FPS 30, forceSetTimeOut: true, resolution: 1 을 TV 에서만 활성화. forceSetTimeOut 은 requestAnimationFrame 대신 setTimeout 으로 강제 휴식을 만들어 webOS 브라우저의 GPU 점유를 줄이는 결정적 트릭. (commit e6d695a: 해상도 절반, 압축 텍스쳐 → png, 프레임 30 제한)

에셋 로딩 분할 + 동적 import

  • src/loaders/gameLoader.ts — getFXPath/getFrameConfig 로 TV 는 이펙트 스프라이트시트를 1/2 해상도 (64×64, 21×54…) 로 자동 치환.
  • vite.config.ts — manualChunks 로 vendor_phaser 를 분리 (Phaser 만 1MB 이상) 하여 초기 로드 차단 시간 단축.
  • src/main.ts — await import('./game') 로 게임 엔진을 동적 import 해서, 초기 HTML 파싱과 webOS 시스템 readiness 확인을 엔진 로딩과 병렬화.
  • src/scenes/LoadingScene.ts — TV 는 팩 사이 1000ms 휴식, 모바일은 50ms 로 차등 → TV 의 메인스레드 stall 회피.

메모리 풋프린트 관리

  • src/managers/StageManager.ts — Endless 모드에서 TV 는 이미 로드된 chapter atlas 중에서만 랜덤 선택해 텍스처 메모리 누적 차단 (모바일은 1~13 자유 랜덤).
  • src/utils/GameObjectPool.ts — Bubble/FX/Tween 모두 풀링, release() 에서 tweens.killTweensOf(obj) 로 dangling tween 까지 정리.
  • src/managers/SoundManager.ts:128-141 — 동시 사운드 10개 cap. WebAudio 풀 폭주로 인한 webOS audio crash 방지.
  • src/managers/SoundManager.ts:34-42 — visibilitychange 로 백그라운드 시 mute (Toss 정책 + 배터리 절감).

3. 빌드 시스템 및 배포 자동화

단일 코드베이스 → 4개 타겟 분기

package.json:

  • build:web → vite build (PC/모바일 웹)
  • build:tv → TARGET_ENV=tv vite build (LG webOS .ipk 패키징용)
  • build:toss → TARGET_ENV=toss vite build --outDir dist-toss
  • build:android → TARGET_ENV=android vite build --outDir dist-android (→ Capacitor sync)
  • cross-env 로 OS-agnostic 환경변수 주입.
  • vite.config.ts:13-16 에서 define 으로 빌드 타임에 VITE_TARGET_ENV 상수 치환 → 사용하지 않는 플랫폼 코드는 트리쉐이킹.
  • postbuild:tv/postbuild:toss 로 dev 전용 docs/editor 디렉토리 자동 제거.

멀티 SDK 통합

  • Vite (웹/번들링) + Capacitor 7 (capacitor.config.ts 로 Android 네이티브 래핑) + Granite (granite.config.ts 로 Toss 미니앱 빌드) — 세 SDK 가 같은 dist* 산출물을 소비하도록 정렬.
  • Capacitor 가 dist-android 를, Granite 가 dist-toss 를, ares-package 가 dist 를 각각 패키징.

webOS 24 호환 레거시 파이프라인

vite.config.ts — @vitejs/plugin-legacy (targets: ['chrome 79']) + 커스텀 'webos-24-fix' 플러그인 자체 작성. modern script 태그를 HTML 후처리로 제거, nomodule 속성 제거로 webOS 24 가 legacy chunk 를 단일 실행하도록 강제. (commit ceb95d4: legacy 문법 처리)

4. 플랫폼 승인 및 컴플라이언스 대응

Tellurium (LG 자동화 테스트) 회피

src/main.ts — LG 심사장의 Tellurium 자동화 도구가 가짜 Tellurium 객체를 주입해 비결정적 race condition 유발. getStatus 메서드 존재 여부로 DevMode 빈 객체 vs 실제 자동화 환경을 구분, 자동화 환경에서만 3000ms dwell time 을 두는 전용 코드 패스로 안정화 (commit 97969c5).

webOS 24 Chrome 79 호환

  • 위 §3 의 webos-24-fix 플러그인이 핵심.

TV 환경의 IAP 부재 — 자체 결제 인프라 구축

src/utils/QrCodePopup.ts — webOS 는 인앱결제 SDK 가 없어 자체 서버 (bellygom.macovill.com/payment/bubblepop) 에 Xsolla 연동, QR 코드 생성 후 3초 폴링으로 결제 완료 감지. TV 사용자 결제 경로를 모바일 웹으로 위임하는 cross-device 결제 플로우.
